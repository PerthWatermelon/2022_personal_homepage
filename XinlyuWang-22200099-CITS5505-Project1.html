All technologies are used for sending executable software code from a server to a client upon request from the client's computer are

code on demand

The executable software code is stored on the web servers.

After receiving the request, the state of the executable software code will change from inactive to active.

The code will be sent to the client's browser based on HTTP.

When the page is displayed, the code will execute locally.

Code on demand is a subset in the field of code mobility.

Single Page Applications

The single page application (SPA) is a web application or website that interacts with the user by dynamically rewriting the current page, rather than loading entire new pages from the server.

The server will send all the raw materials, including HTML, CSS, and JS, of the page when first time clients open it on their devices. Then all the materials will be processed on the client's device.

The SPA removes the need for constant reloading of the full webpage, and it only dynamically reloads the relevant content for each user interaction.

Page pulls data through ajax to achieve response function

Since the functionality of SPAs is handled within the browser, this type of application is always made using frontend frameworks like React, Angular, Vue etc.

SPA brings a faster user experience and needs fewer server resources.

Example of application: Gmail, Google Maps, Facebook and Twitter etc.

Pros (click to see the content)
  • Faster User Experience (after first load):

    Communicate with the server to maintain data without refreshing the entire page. This makes the Web application more responsive to user interaction and avoids sending unchanged information over the network, reducing user wait time and bringing a fast user experience.

  • Fewer Server Resources

    Code on demand can transfer some workload of the server to the client, which will reduce the burden on the server and bandwidth, and save space and broadband rental costs.

  • Separating Content and Presentation

    It is conducive to the cooperation of the development team, reduces errors when web applications modify page content, and improve efficiency.

Cons (click to see the content)
  • Broken browser mechanism:

    In the case of dynamically updating the page, the user cannot go back to the previous page state, because the browser can only remember the static page in the history.

  • SEO Difficulties:

    Each page of the site comes from a URL. This structure may limit the functionality that the SPA can use to take advantage of search engine optimization (SEO) techniques.c

  • Security Issues:

    Exposes the details of how the browser interacts with the server. It is difficult to avoid some known security weaknesses, such as cross-site footstep attacks, SQL injection attacks, and Credentials-based security vulnerabilities.

Code on demand is goog but...

It has many constrains

Client-Server

Code on demand needs communication between clients and servers. The client focus on the front-end and the servers' concern is about back-end stuff. So if the network does not include both of them, we cannot use code on demand.

Stateless

States of the sender will be sent in TCP. But it will make communication complicated. HTML chooses the stateless communication method. To make the code from a server can work smoothly on a client's browser, every code on demand request should include all the necessary information to perform the request.

Cache

The client can cache some infrequently changed information locally. The information in the cache can be obtained directly and does not need to send a request to severs again. If the client's cache is available, the server does not need to send a complete response, which can reduce the pressure on the server.

Cod

Code on demand is the only optional constraint in REST. To improve flexibility in clients, the server will decide how the code works. That will reduce visibility. After the client executes the code from the server, the client does not know what information will be collected. That will cause a big problem, for example, when users input their passwords for their bank accounts may be stolen by malicious software.
